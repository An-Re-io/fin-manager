# Задача
Спроектировать небольшое функциональное приложение, нацеленное на удобный учет повседневных расходов.

# Функциональные требования:
Пользователь должен иметь возможность
1. Создать "Статью расхода" (их может быть несколько). Статья расхода состоит из названия, общей суммы (в месяц) выделенной на эту статью расходов, и суммы
  расходов (т.е. сумма покупок в месяц). Статей расхода может быть несколько.

2. Пользователь должен иметь возможность выбрать статью расхода и ввести конкретную покупку, т.е. задать сумму и описание, после чего транзакция добавляется в общую сумму расходов на конкретную статью расходов

# Пользовательский интерфейс
1. Должен быть ориентирован в первую очередь на использование на смартфоне, адаптирован для использование с помощью жестов и тапов. Иметь лаконичный, но информативный интерфейс
2. При входе в приложение, пользователь видит главную страницу со списком статей расходов, каждая из которых представляет из себя горизонтальную шкалу, которая отображает отношение общей суммы выделенной на эту статью расхода к сумме затраченных денег на эту статью расхода
А так же название к текущей статье расхода
3. Пользователь должен иметь возможность вводить расход как в волюте Сербские динары, Евро, Русские Рубли, но основная валюта, в которую происходит конвертация это динары.
Курсы валют должны автоматически подтягиваться из открытых источников, но на первом этапе их можно задать вручную, в настройках приложения
4. При нажатии пользователем на конкретную статью расходов из списка пользователь попадает на страницу с подробным списком транзакций, где каждая транзакция это цена траты + описание траты
